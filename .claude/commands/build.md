## 知识沉淀 Build 指令

请对指定的 Markdown 文件进行整理、索引更新，并发布到 GitHub Pages。

### 输入

$ARGUMENTS

> 格式：`<文件路径> [简短描述]`
> 示例：`小程序笔记/skyline&webview.md 小程序渲染引擎对比与演进`

### 执行步骤

请严格按照以下步骤执行：

#### 第一步：整理 Markdown 内容

1. 读取目标 Markdown 文件
2. 将内容整理为结构良好的 Markdown 格式：
   - 添加合适的一级标题（`#`）和多级子标题
   - 将散乱的文本整理为有序/无序列表
   - 对关键术语和概念使用**加粗**标记
   - 代码片段使用代码块包裹，并标注语言类型
   - 表格数据使用标准 Markdown 表格格式
   - 确保段落之间有合理的空行
   - 保持原有内容的准确性，不增减核心信息
3. 写回整理后的内容

#### 第二步：更新 `_sidebar.md`

1. 读取 `_sidebar.md`
2. 根据文件路径判断所属分类（如 `小程序笔记/xxx.md` → 归入「小程序笔记」分类）
3. 如果该分类已存在，在分类下添加新条目；如果分类不存在，创建新分类并添加条目
4. 如果该文件条目已存在，则跳过不重复添加
5. 条目格式：`  - [标题](文件路径)`

#### 第三步：更新 `README.md`

1. 读取 `README.md`
2. 在对应分类的「内容目录」下添加新条目
3. 如果该文件条目已存在，则跳过不重复添加
4. 如果分类不存在，在 `## 📚 内容目录` 下新增 `### 分类名` 和对应条目
5. 条目格式：`- [标题](文件路径) - 简短描述`

#### 第四步：Git 提交并推送

1. 执行 `git add -A`
2. 执行 `git commit -m "docs: 添加/更新 <文件名>"` （commit message 根据实际情况调整）
3. 执行 `git push` 推送到远程仓库，更新 GitHub Pages

### 注意事项

- 使用中文回答和交互
- 整理内容时保持原始信息完整，只优化格式和结构
- 如果输入没有提供简短描述，请根据文件内容自动生成一句话描述
- 如果文件路径所属分类是新的（`_sidebar.md` 和 `README.md` 中都没有），需要同时在两个文件中创建该分类
